# FuckBerry — Комплексная система управления Wildberries

> **📋 ВАЖНО ДЛЯ РАЗРАБОТЧИКА**: Перед началом работы с проектом обязательно прочитайте [DEVELOPMENT_LOG.md](DEVELOPMENT_LOG.md) для понимания контекста и истории изменений!

Многофункциональное Flask-приложение для полного цикла работы с маркетплейсом Wildberries: от аналитики заказов и продаж до управления поставками FBS/FBW, генерации этикеток и финансовой отчетности. Включает систему авторизации, уведомлений и автоматизации бизнес-процессов. 

## 🚀 Основные возможности

### 📊 Аналитика и отчетность
- **Аналитика заказов**: графики по дням, сводка по складам, ТОП товаров
- **Финансовые отчеты**: детальная аналитика доходов и расходов с экспортом в Excel
- **Отчеты по продажам**: статистика по складам с возможностью фильтрации
- **Экспорт данных**: заказы, отчеты и аналитика в формате Excel (xlsx)

### 🏪 Управление маркетплейсом FBS
- **Заказы FBS**: получение новых заданий на сборку с сортировкой по дате
- **Остатки на складах FBS**: мониторинг запасов с автоматическим обновлением
- **Создание поставок**: добавление заданий в существующие или новые поставки
- **Автоматизация остатков**: обновление через Excel-файлы с отслеживанием изменений

### 📦 Управление поставками FBW
- **Список поставок**: детальная информация о поставках с количеством коробов
- **Детали поставок**: товары и упаковки в каждой поставке
- **Статус доставки**: отслеживание товаров и коробов в пути

### 🏷️ Инструменты и автоматизация
- **Генерация этикеток**: создание этикеток для коробов с реквизитами отправителя
- **Коэффициенты приёмки**: сетка коэффициентов на 14 дней с цветовой индикацией
- **Система уведомлений**: уведомления о новых заказах FBS и обновлениях сервиса
- **Кэширование данных**: оптимизация производительности для всех модулей

### 👥 Система пользователей
- **Авторизация**: система входа с проверкой срока действия подписки
- **Управление пользователями**: создание, блокировка, настройка прав доступа
- **Профили пользователей**: управление API-токенами и реквизитами отправителя
- **Административная панель**: управление пользователями и настройками системы

### 🔄 Технические особенности
- **Асинхронное обновление**: обновление данных без перезагрузки страницы
- **Кэширование**: оптимизация запросов к API Wildberries
- **Адаптивный интерфейс**: современный UI с Bootstrap 5
- **Московский часовой пояс**: корректное отображение времени

## 📋 Требования

### Системные требования
- **Python**: 3.10+ (рекомендовано 3.11+)
- **База данных**: SQLite (встроенная) или PostgreSQL/MySQL для продакшена
- **Память**: минимум 512MB RAM
- **Дисковое пространство**: 100MB для приложения + место для кэша

### API и доступы
- **Wildberries API токен**: Supplier Statistics API для аналитики
- **Marketplace API токен**: для работы с FBS/FBW
- **Supplies API токен**: для управления поставками

### Первоначальная настройка
- Создание администратора: `admin/admin` (изменить пароль после первого входа)
- Настройка API токенов в профиле пользователя
- Настройка реквизитов отправителя для генерации этикеток

## 🚀 Установка и запуск

### Установка зависимостей
```bash
# Создание виртуального окружения
python -m venv .venv

# Активация виртуального окружения
# Windows PowerShell
.\.venv\Scripts\Activate.ps1
# Linux/macOS
source .venv/bin/activate

# Установка зависимостей
pip install -r requirements.txt
```

### Первый запуск
```bash
# Установка переменных окружения (опционально)
# Windows PowerShell
$env:FLASK_SECRET = "your-random-secret"
$env:FLASK_ENV = "development"

# Linux/macOS
export FLASK_SECRET="your-random-secret"
export FLASK_ENV="development"

# Запуск приложения
python app.py
```

### Доступ к приложению
- **URL**: http://127.0.0.1:5000
- **Логин по умолчанию**: `admin` / `admin`
- **Порт**: 5000 (настраивается в app.py)

## 🎯 Первые шаги

### 1. Авторизация и настройка
1. Войдите в систему с логином `admin` / `admin`
2. Смените пароль администратора в профиле
3. Создайте пользователей через админ-панель (если нужно)
4. Настройте API токены в профиле пользователя

### 2. Настройка API токенов
1. Перейдите в **Профиль** → **API токены**
2. Добавьте токены для:
   - **Supplier Statistics API** (для аналитики заказов)
   - **Marketplace API** (для FBS/FBW)
   - **Supplies API** (для поставок)

### 3. Настройка реквизитов отправителя
1. Перейдите в **Профиль** → **Реквизиты отправителя**
2. Заполните данные для генерации этикеток:
   - Наименование отправителя
   - Адрес
   - Контактное лицо

### 4. Работа с аналитикой
1. Откройте **Аналитика заказов**
2. Выберите период и нажмите **Загрузить**
3. Изучайте графики, таблицы и экспортируйте данные в Excel

### 5. Управление FBS
1. Перейдите в **Маркетплейс FBS** → **Заказы FBS**
2. Просматривайте новые задания на сборку
3. Создавайте поставки и добавляйте задания
4. Мониторьте остатки на складах FBS

### 6. Дополнительные функции
- **Поставки FBW**: управление поставками на склад WB
- **Финансовые отчеты**: детальная аналитика доходов
- **Генерация этикеток**: создание этикеток для коробов
- **Система уведомлений**: мониторинг новых заказов

## 🔗 API Эндпоинты

### Основные страницы
- `GET /` → редирект на `/orders`
- `GET|POST /orders` — аналитика заказов и продаж
- `GET|POST /fbs` — заказы FBS и управление поставками
- `GET /fbs-stock` — остатки на складах FBS
- `GET /fbw` — поставки FBW
- `GET /report/finance` — финансовые отчеты
- `GET /report/sales` — отчеты по продажам
- `GET /marketing` — маркетинг и продвижение (рекламные кампании WB)
- `GET /marketing/campaign/<id>` — карточка рекламной кампании
- `GET /tools/labels` — генерация этикеток
- `GET /profile` — профиль пользователя
- `GET /admin/users` — управление пользователями (только админы)

### API для данных
- `POST /api/orders-refresh` — обновление аналитики
- `POST /api/fbs/tasks` — получение заданий FBS
- `POST /api/fbs-stock/refresh` — обновление остатков FBS
- `GET /api/fbw/supplies` — список поставок FBW
- `GET /api/report/finance` — данные финансового отчета
- `GET /api/marketing/data` — рекламные кампании и статистика за период
- `GET /api/marketing/campaign/<id>` — детализация кампании (KPI, товары, фразы)
- `GET /api/top-products-orders` — ТОП товаров по заказам

### Экспорт данных
- `POST /export` — экспорт заказов в Excel
- `GET /report/finance/export` — экспорт финансового отчета
- `POST /fbs/export` — экспорт заданий FBS
- `POST /tools/labels/download` — генерация этикеток

### Авторизация
- `GET|POST /login` — вход в систему
- `GET /logout` — выход из системы

## 🐛 Решение проблем

### Частые ошибки
- **HTTP 429 (Too Many Requests)**: сузьте период запроса; приложение автоматически повторяет запросы с задержкой
- **Пустые графики/таблицы**: нажмите кнопку «Загрузить» для обновления кэша
- **Ошибка авторизации**: проверьте правильность логина/пароля
- **Нет данных FBS**: убедитесь, что настроен Marketplace API токен

### Отладка
- Проверьте логи приложения в консоли
- Убедитесь, что все API токены корректны
- Проверьте подключение к интернету
- Очистите кэш через перезапуск приложения
